# Partner Strategy Workflow Copilot

Internal tooling for the Partner Strategy team. Turns onboarding data-gap anomalies
into routed, tracked, and drafted customer outreach so specialists spend less time
triaging and chasing, and more time resolving.

## What it does

During portfolio onboarding, the platform flags data gaps (missing utility bills,
unmatched meters, incomplete equipment inventory). This tool takes that anomaly report
and, for each gap:

- routes it to the right building, account, owner, and severity
- creates a tracked item so nothing is lost across a large portfolio
- drafts the outreach to the responsible party at the customer
- holds the draft for specialist review and approval before anything is sent

The specialist reviews and approves. They don't start from a blank page, and they don't
lose track of what's outstanding against the 30-day onboarding SLA.

## What it does not do

It does not detect, validate, or estimate data — the platform already does that. This
tool operates on the anomaly report the platform produces. It is a workflow layer on
top, not a data-quality engine.

## How it works

- **Deterministic:** routing, tracking, state, scheduling, and all send-safety checks
  are handled in code.
- **LLM-assisted:** gap summaries and outreach drafts are generated by Claude, behind
  a provider interface. Drafts are constrained to reference only the gaps present in
  the source report, and always require human approval before sending.

## Stack

Python · Django REST Framework · Celery · PostgreSQL · Claude

Designed to run on AWS in future iterations: Celery on SQS, API and workers on Fargate, infrastructure via CDK.
